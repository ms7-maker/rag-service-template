# Публичный шаблон и приватная продакшен-версия

Этот репозиторий (`rag-service-template`) — **упрощённый публичный каркас**.  
Продакшен-логика (маршрутизация, fallback, боевые промпты) **не публикуется** и живёт в отдельном приватном репозитории.

---

## Что есть в публичном шаблоне

| Компонент | Статус | Описание |
|-----------|--------|----------|
| Подготовка документов | ✅ публично | `cleaner.py`, `render_and_clean.py`, `data_paths.py` |
| Векторный поиск | ✅ публично | `vector_store.py` (FAISS, чанки, эмбеддинги) |
| Линейный RAG-пайплайн | ✅ публично | `rag_pipeline.py`: кеш → поиск → LLM → кеш |
| Базовый кеш | ✅ публично | `cache.py` (SQLite, хеш запроса) |
| Чат-интерфейс | ✅ публично | `app.py` |
| Оценка качества | ✅ публично | `evaluate_ragas.py` |
| Промпты | ⚠️ только заглушки | `prompts.py` — учебные шаблоны, не боевые |
| Маршрутизация | ⚠️ заглушка | `routing.py` — всегда `rag`, без классификации намерений |
| Fallback | ⚠️ заглушка | `fallback.py` — нет цепочки запасных сценариев |

---

## Что остаётся в приватном репозитории

| Компонент | Почему приватно |
|-----------|-----------------|
| **Архитектура пайплайна** | Полная схема: intent → router → retriever → rerank → generator → post-check |
| **Промпты** | Системные инструкции, few-shot, доменные правила, тон бренда |
| **Маршрутизация** | Классификация запросов (FAQ / цены / поддержка / off-topic), выбор индекса и модели |
| **Кеш (расширенный)** | TTL, инвалидация по версии документов, сегментация по ролям |
| **Fallback** | Низкая уверенность retrieval → уточняющий вопрос → эскалация → запасная модель |

Приватный репозиторий подключается как **drop-in замена** модулей `prompts.py`, `routing.py`, `fallback.py` и расширенной версии `rag_pipeline.py`.

---

## Рекомендуемая схема двух репозиториев

```text
rag-service-template (public, этот репо)
├── Учебный каркас под проприетарной лицензией автора
├── Документация и onboarding
└── Заглушки расширений (prompts, routing, fallback)

rag-service-core (private)
├── prompts_production.py
├── routing.py          — intent + выбор коллекции
├── fallback.py         — цепочка запасных ответов
├── rag_pipeline.py     — полный пайплайн
└── cache_advanced.py   — TTL, versioning, role-aware cache
```

Публичный шаблон можно форкать для обучения.  
Приватное ядро — для коммерческих и корпоративных внедрений.

---

## Что делать при развитии проекта

1. **Сейчас (public):** оставить линейный пайплайн как есть — он уже безопасен для публикации.
2. **Приватно:** развивать routing, fallback и промпты только в `rag-service-core`.
3. **Синхронизация:** из public периодически переносить в private только инфраструктурные улучшения (cleaner, vector_store), но не наоборот.
4. **Лицензия:** public-репозиторий под проприетарной лицензией автора (см. `LICENSE`); форки и коммерция — только с письменного разрешения.
